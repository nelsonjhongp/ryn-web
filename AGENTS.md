# AGENTS.md

## Preferencias de este proyecto

- El inicio debe ser corto, directo y centrado en el contenido.
- Evitar textos técnicos, explicaciones de diseño o bloques informativos en la portada.
- Priorizar una presentación emocional, simple y bonita.
- Cuando sea posible, usar accesos visuales suaves como círculos en vez de tarjetas rectangulares.
- Mantener una apariencia luminosa y agradable en celular.
- Dar prioridad a una experiencia cómoda en móvil, con elementos fáciles de tocar.
- No cambiar el estilo de las páginas internas salvo que se pida explícitamente.
- `index.html` debe quedarse como entrada breve hacia las demás páginas.
- Las páginas distintas de `index.html` deben vivir dentro de `pages/`.
- Usar nombres de archivos en formato web con guiones medios (`kebab-case`) para nuevas páginas cuando sea razonable.
- Si una nueva página trae CSS o JS propio, integrarlo en `css/` y `js/` en vez de dejarlo embebido dentro del HTML, siempre que no complique innecesariamente la página.
- En páginas especiales, priorizar interacciones bonitas y claras antes que controles duplicados o texto de estado separado.
- Evitar movimientos innecesarios del panel principal; si hay animación, debe concentrarse en detalles decorativos o personajes.
- Si un control puede resolverse con un icono claro dentro del panel, preferir eso antes que botones grandes con texto extra.
